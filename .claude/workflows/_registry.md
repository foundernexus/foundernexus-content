# fnx Dynamic Workflows Registry

Saved Dynamic Workflow scripts for large / fan-out / audit / cross-checked-research work — the tier
where the orchestrator coordinates subagents in code (and background subagents can nest). Each workflow
carries a Performance Outcomes rubric.

Candidate workflows for this repo:
- **substrate-consistency-audit** — fan out over `substrate/*.md`, cross-check terminology, lane names,
  and CTA maps for contradictions; synthesize a fix list.
- **atomization-fan-out** — one approved draft → N social derivatives (per `campaign-lanes.md` voice),
  each verified against `content-standards.md`.
- **keyword-cluster-research** — multi-source research to propose/validate new `keywords.yaml` clusters,
  each adversarially checked against the lane gate before proposal.
